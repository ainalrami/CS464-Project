# Report Notes — Output Artifacts Guide

This file explains the output artifacts generated by the EuroSAT ML/DL comparison study.

---

## `results/metrics/`

| File | Description |
|------|-------------|
| `ml_summary.csv` | Summary of all classical ML models: CV score, val/test accuracy and F1 for each feature mode × model combination. |
| `dl_summary.csv` | Summary of DL experiments: with/without augmentation, val/test accuracy and F1. |
| `overall_test_summary.csv` | All models ranked by test accuracy — the main comparison table. |
| `robustness_results.csv` | Performance of each model under each degradation condition (clean, blur, noise, downsample). |
| `robustness_pivot.csv` | Pivot table of robustness results — models as rows, conditions as columns. |
| `*_test_report.csv` | Detailed per-class precision/recall/F1 for each model on the test set (sklearn classification_report format). |
| `*_test_metrics.csv` | Single-row summary with accuracy, macro P/R/F1 for each model on test set. |
| `*_val_*.csv` | Same as above but for the validation set. |

---

## `results/confusion_matrices/`

PNG heatmaps showing the confusion matrix for each model on each split.
- Rows = true labels, columns = predicted labels.
- Higher diagonal values = better performance.
- Use these to identify which classes are commonly confused.

---

## `results/plots/`

| File | Description |
|------|-------------|
| `*_training_curves.png` | Loss and accuracy curves for the DL model (train vs. validation over epochs). Shows whether the model converged and if overfitting occurred. |
| `model_comparison.png` | Bar chart comparing accuracy and macro F1 across all models on the test set. |
| `per_class_f1_comparison.png` | Grouped bar chart showing F1 score per class for each model — useful for identifying class-specific strengths/weaknesses. |
| `robustness_comparison.png` | Grouped bar chart showing how each model's accuracy degrades under different corruption types and severity levels. |

---

## `results/models/`

| File | Description |
|------|-------------|
| `*.pkl` | Saved sklearn pipelines (StandardScaler + Classifier) for ML models. Load with `joblib.load()`. |
| `*_best.pt` | PyTorch state dict for the best DL model checkpoint (based on validation loss). Load with `torch.load()`. |

---

## `results/logs/`

| File | Description |
|------|-------------|
| `split_train.csv`, `split_val.csv`, `split_test.csv` | The exact train/val/test split — image paths and labels. Ensures reproducibility across runs. |
| `class_names.json` | Mapping from class name to integer label index. |
| `*_training_log.csv` | Per-epoch training metrics (loss, accuracy, LR) for the DL model. |

---

## How to Interpret Results

1. **Start with `overall_test_summary.csv`** — see which model achieves the best accuracy/F1 on the clean test set.
2. **Check `robustness_pivot.csv`** — see how each model degrades under corruptions. A good model maintains high accuracy under increasing severity.
3. **Look at `per_class_f1_comparison.png`** — some classes (e.g., Highway vs. River) may be harder for certain models.
4. **Review confusion matrices** — find the most confused class pairs.
5. **DL training curves** — verify convergence and absence of severe overfitting.
